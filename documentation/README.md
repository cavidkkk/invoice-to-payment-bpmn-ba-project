# Documentation

This folder contains the main project documentation for the Invoice-to-Payment Business Analysis portfolio project.

The documents in this folder explain the business context, process scope, business requirements, functional requirements, BPMN model summary, data properties, risks, controls, KPIs, and Jira-style user stories related to the Invoice-to-Payment process.

## Files Included

| File                                          | Purpose                                                                                                                                                                           |
| --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Invoice_to_Payment_BA_Portfolio_Project.pdf` | Main portfolio presentation PDF that summarizes the full Business Analysis project.                                                                                               |
| `BRD_Invoice_to_Payment.pdf`                  | Business Requirements Document covering the business problem, objectives, scope, stakeholders, business requirements, business rules, risks, controls, and KPIs.                  |
| `FRD_Invoice_to_Payment.pdf`                  | Functional Requirements Document covering system behavior, functional requirements, validation rules, exception handling, status updates, notifications, and acceptance criteria. |

## Documentation Purpose

The purpose of this documentation is to show how a Business Analyst can move from process understanding to structured requirements documentation.

This folder supports the project by explaining:

* Why the Invoice-to-Payment process matters
* What business problem the process solves
* Which stakeholders are involved
* What the business requirements are
* How the system should behave
* Which data fields are required
* What risks and controls should be considered
* Which KPIs can measure process performance
