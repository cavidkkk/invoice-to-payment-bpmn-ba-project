# SQL Analysis

This folder will contain SQL queries for analyzing the synthetic Invoice-to-Payment dataset.

The SQL analysis will focus on process validation, exception detection, payment control, and KPI calculation.

Planned files:

| File                             | Purpose                                                          |
| -------------------------------- | ---------------------------------------------------------------- |
| `invoice-validation-queries.sql` | SQL queries for invoice validation checks.                       |
| `exception-analysis.sql`         | SQL queries for analyzing invoice exceptions and process issues. |
| `kpi-queries.sql`                | SQL queries for calculating Invoice-to-Payment KPIs.             |

## Planned Analysis Areas

The SQL analysis will include:

* Duplicate invoice detection
* Blocked vendor checks
* Missing PO checks
* PO / invoice amount mismatch checks
* Pending approval analysis
* Late payment analysis
* Payment failure analysis
* Reconciliation exception analysis
* Invoice exception rate
* First-time match rate
* Payment success rate
* Reconciliation success rate

## Purpose

This folder extends the Business Analysis project by showing how process documentation can be connected with data analysis and operational reporting.
